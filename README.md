# E-Commerce Recommendation & Revenue Impact Analysis

A full end-to-end data science and AI project built on the UCI Online Retail dataset. The project combines business analytics, machine learning recommender systems, and large language model integration to demonstrate how personalised product recommendations can be quantified in terms of revenue impact.

---

## Project Overview

This project covers the complete data pipeline from raw transaction data to a production-ready recommendation API with AI-powered explanations. It was built to simulate how a mid-size e-commerce retailer could implement and measure the impact of a recommendation system without enterprise-level infrastructure.

**Core components:**
- Customer segmentation using RFM (Recency, Frequency, Monetary) analysis
- Three recommender systems: rule-based, collaborative filtering, and content-based
- A/B test framework to quantify revenue impact by customer segment
- LLM integration using Anthropic Claude for natural language recommendation explanations and a conversational shopping assistant
- REST API serving all three recommenders in real time
- Tableau dashboard for business stakeholder communication

## Recommender Systems

### Rule-Based
Uses co-purchase frequency analysis. Products frequently bought together are recommended to customers who haven't purchased them yet. Fast, interpretable, and pushes proven bestsellers.

### Collaborative Filtering
Uses a user-item matrix with KNN cosine similarity to find customers with similar purchase patterns. Recommends products bought by similar customers. More personalised and diverse than rule-based.

### Content-Based
Uses TF-IDF vectorisation on product descriptions with cosine similarity to find semantically similar products. Most diverse approach with 2,036 unique products recommended across all customers.

### AI Layer
All recommendations from the `/recommend/all/` endpoint include a natural language explanation generated by Claude, describing why the products were recommended based on the customer's purchase history. A conversational `/chat` endpoint allows customers to describe what they are looking for in natural language and receive personalised suggestions.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/` | API homepage with route documentation |
| GET | `/recommend/rule/<customer_id>` | Rule-based recommendations |
| GET | `/recommend/collaborative/<customer_id>` | Collaborative filtering recommendations |
| GET | `/recommend/content/<customer_id>` | Content-based recommendations |
| GET | `/recommend/all/<customer_id>` | All three methods with AI explanation |
| POST | `/chat` | Conversational shopping assistant |

### Running the API locally

```bash
# Clone the repo
git clone https://github.com/NeeharSatti1998/ecommerce-recommendation-revenue-analysis.git
cd ecommerce-recommendation-revenue-analysis

# Create virtual environment
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Mac/Linux

# Install dependencies
pip install -r api/requirements.txt

# Add your Anthropic API key
echo "ANTHROPIC_API_KEY=your-key-here" > api/.env

# Run the API
cd api
python app.py
```

### Example API call

```bash
# Get all recommendations with AI explanation
curl http://127.0.0.1:5000/recommend/all/17850

# Conversational chatbot
curl -X POST http://127.0.0.1:5000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "I am looking for a gift under 20 pounds", "customer_id": 17850}'
```
